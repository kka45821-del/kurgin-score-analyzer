# KURGIN v1.11.7 Visual / Market Validation Set

Date: 2026-05-29T01:49:48Z

## Purpose

This stage validates whether Candidate v2 class-cap logic is visually and commercially reasonable.

It does not change official KURGIN Score.

## Why this stage exists

Regression showed Candidate v2 is conservative. But before promotion we need real-world validation:

- Do cap cases actually look less suitable for Elite/Premium/High?
- Are controls with high score visually strong?
- Are diameter warnings meaningful to jeweler/customer perception?
- Are measurement problems data artifacts or real visual issues?

## How to use the workbook

File:

```text
kurgin_v1_11_7_visual_market_validation_template.xlsx
```

Sheets:

- `Summary` — purpose and decision context.
- `Review Queue` — cases selected for review.
- `Validation Form` — fill this manually after checking photo/video.
- `Rubric` — how to rate visual and market qualities.
- `Decision Guide` — how to decide if Candidate cap is right.
- `Lists` — dropdown values.

## Required reviewer inputs

For each case:

- Photo URL
- Video URL
- Visual Size Rating
- Brightness Rating
- Fire Rating
- Pattern/Symmetry Rating
- Eye Clean / Inclusions
- Milky/Shade Observation
- Marketability
- Jeweler Decision
- Candidate Cap Decision
- Recommended Public Class
- Notes

## Decision rule

Candidate v2 can move toward promotion only if:

1. Most cap cases receive `Candidate Cap Decision = Agree`.
2. High-score controls do not show false concern.
3. Cases marked `Needs more data` are resolved with better media.
4. Experts can explain why class cap is reasonable.

## Recommended next stage

```text
v1.11.8 Visual Validation Results Analysis
```

After the workbook is filled, analyze it and decide:

- keep Candidate v2 as preview only;
- use public class cap;
- revise class cap rules;
