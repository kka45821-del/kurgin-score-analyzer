# FINAL ANALYZER EXCEL OUTPUT CONTRACT v0.1

Repo: `kka45821-del/kurgin-score-analyzer`
Scope: final Analyzer Excel output contract for future Admin/Catalog import.
Mode: docs only. Formula, Excel generator, PDF/report generator and production code are not changed by this document.

This document defines the canonical Excel output contract for the future pipeline:

```text
Analyzer → final Excel → Admin import → Public Catalog
```

## 1. Purpose of the final Excel

The Analyzer must eventually produce one canonical Excel file for downstream catalog operations.

The file must contain:

- one canonical sheet for Admin import;
- controlled Analyzer result sheets;
- validation and diagnostics sheets for review;
- a schema/readme sheet describing the contract.

The final Excel must separate public-safe catalog data from internal Analyzer diagnostics.

The Admin/Catalog pipeline must treat `PUBLIC_CATALOG_IMPORT` as the only public import sheet. All other sheets are supporting or internal and must not be imported into the public catalog directly.

## 2. Required sheets

The final Excel file must contain exactly these required sheets:

1. `PUBLIC_CATALOG_IMPORT`
2. `ANALYZER_RESULTS`
3. `VALIDATION_ERRORS`
4. `INTERNAL_DIAGNOSTICS`
5. `README_SCHEMA`

Additional sheets may be added later only through a new version of this contract.

## 3. Sheet: PUBLIC_CATALOG_IMPORT

Purpose: canonical public-safe sheet read by Admin import.

Admin must read this sheet and ignore internal diagnostic sheets for catalog publication.

### PUBLIC_CATALOG_IMPORT rules

- Must not include raw formula internals.
- Must not include weights, thresholds, diagnostics or debug traces.
- Must not claim certification/appraisal.
- Must not imply order, reserve, payment or guaranteed price effect.
- Must be importable without reading any internal sheet.

### Validation error examples

| Error code | Severity | blocks_catalog_import | Meaning |
|---|---|---|---|
| `missing_required_identity` | blocker | yes | Missing both stable ID and report number. |
| `missing_shape` | blocker | yes | Shape missing. |
| `unsupported_shape` | review | no by default | Shape not supported by Analyzer scoring. |
| `missing_carat` | blocker | yes | Carat missing. |
| `invalid_numeric_geometry` | blocker | yes | Required geometry cannot be parsed. |
| `missing_geometry` | review | no by default | Row can exist in admin but not as scored catalog row. |
| `score_unavailable` | review | no by default | Analyzer did not calculate score. |
| `source_conflict` | review | no by default | Conflicting source values. |
| `missing_price_state` | blocker | yes | `price_status` or `public_action` absent. |

## 9. Validation gate

### Block PUBLIC_CATALOG_IMPORT publication

A row must be blocked from public catalog publication when:

- required identity is missing;
- required display fields are missing (`shape`, `carat`, `color`, `clarity`);
- numeric fields required for the selected publication state are invalid;
- `validation_status = blocked`;
- `show_in_catalog` is not explicitly true;
- `price_status` is missing;
- `public_action` is missing;
- data source is too incomplete for safe display.

### Require `admin_review_required = true`

A row must require admin review when:

- geometry is incomplete but public display is otherwise possible;
- Analyzer score is unavailable;
- shape is unsupported by current scoring model;
- data source has conflicts;
- visual check is recommended;
- critical risk is true;
- report quality is uncertain;
- public summary/tags need review.

### Allow publication with warning

A row may be published with warning only when:

- required public display fields are valid;
- unsafe internal fields are absent from public import;
- `validation_status = warning` or `ready`;
- `admin_review_required` is false or admin has explicitly approved;
- `show_in_catalog = true`;
- `price_status` and `public_action` are safe public states.

## 10. Admin import assumptions

Admin import must follow these assumptions:

- Admin reads only `PUBLIC_CATALOG_IMPORT` for catalog data.
- Admin ignores `INTERNAL_DIAGNOSTICS`.
- Admin may show `ANALYZER_RESULTS` or `VALIDATION_ERRORS` only in internal review UI.
- Public catalog does not depend on raw formula internals.
- Public catalog does not read raw diagnostics, breakdown or formula configuration.
- If `validation_status != ready`, a row must not be published without review/approval.
- If `show_in_catalog != true`, a row must remain hidden.
- If `price_status` or `public_action` is absent, a row must not be published.
- If `admin_review_required = true`, Admin must require explicit approval before public publishing.
